Hakon 使用说明书

做重构的时候避免不了很多琐碎的操作，css 加前缀、合并雪碧图等等，虽然现在有很多工具可以完成这些工作，像 Autoprefixer 可以一键加前缀，css gaga 可以合并雪碧图，但是之后的修改维护却很费劲，而且现在有很多插件可以供我们使用，只需要把他们结合起来，再给个图形界面，这样操作起来就方便许多（真是越来越懒），所以就让这些无聊琐碎的操作交给 Hakon 处理吧

github地址 : https://github.com/maxuekai/Hakon
有兴趣的欢迎点个星星~

Hakon界面

Hakon 现在分为 “工具”、“模板” 两个部分；“工具” 主要是重构的 “后处理器”，旨在解决重构的最后一步，“模板” 是重构的 “前处理器”，主要让页面迈出第一步

工具

工具部分是 Hakon 用来处理 css 和图片的面板，所有的操作都是通过拖拽文件完成的，现在的主要操作有：

  1. 合并雪碧图 （pc、wap模式）
  2. 压缩图片 （包括雪碧图压缩）
  3. 使用最新的 css （postcss cssnext 功能）
  4. 自动加前缀 （autoprefixer)
  5. 压缩 css （postcss cssnano）
  6. 处理 @import 文件 （将 @import 的 css 文件合并）

注： 除了压缩图片是拖拽 图片文件 ，其他都是拖动 css 文件（包括合并雪碧图）

经过 Hakon 处理后，会在项目目录下生成一个 “dist” 文件夹，处理完的文件就在此文件夹里面，重构需要做的，只是把所有文件处理完后，将 dist 文件夹下的文件提交给开发就 ok 了，而我们在后面的修改维护的时候，只需要维护 dist 之外的文件，修改完之后将文件拖到 Hakon 中处理，将 dist 文件夹下的文件进行更新即可，所以 Hakon 属于重构的 “后处理器”

接下来我们来对比一下 Hakon 处理前后的 css 文件：

  1. 合并雪碧图
    首先，我们需要把要合并雪碧图的图片放到 img 文件夹下的 ‘slice’ 文件夹下，slice 文件夹下可以建多个文件夹（合并多张雪碧图），雪碧图的名字，就是 ‘spr-’ + ‘文件夹名’，slice 当层文件夹下的图片，会合并成一张名为 ‘spr.png’ 的雪碧图，如下图文件目录所示:

    而 css 文件中的引用，也会将雪碧图的路径、定位直接替换，如以下处理前后的 css 代码对比：

    原 css 代码：

    .icon,.exp{
    display: block;
    width: 64px;height: 64px;
}
.icon-gift{
    background: url(../img/slice/icon/icon-gift.png);
}
.icon-home{
    background: url(../img/slice/icon/icon-home.png);
}
.exp01{
    background: url(../img/slice/exp/exp01.png);
}
.exp02{
    background: url(../img/slice/exp/exp02.png);
}

    处理后的 css 代码：

    .icon,.exp{
    display: block;
    width: 64px;height: 64px;
}
.icon-gift{
     background-image: url(../img/spr-icon.png?1527757031305);
     background-position: 0px 0px;
     background-repeat: no-repeat;
 }
 .icon-home{
     background-image: url(../img/spr-icon.png?1527757031305);
     background-position: -69px 0px;
     background-repeat: no-repeat;
 }
 .exp02{
     background-image: url(../img/spr-express.png?1527757031308);
     background-position: 0px -69px;
     background-repeat: no-repeat;
 }

 .exp03{
     background-image: url(../img/spr-express.png?1527757031308);
     background-position: 0px 0px;
     background-repeat: no-repeat;
 }

    注： 很多人会觉得合并雪碧图是将图片拖到 Hakon 里，如果这么处理的话，那就跟 css gaga 没什么不同了，而 Hakon，是直接处理 css，通过查找 css 里引用的图片，将其合并雪碧图，并将图片的定位，写到 css background-position 中，从而减少我们手动测量雪碧图定位，并替换代码的时间，而这也是我之前开发中，最花时间的部分；当我们需要修改雪碧图的时候，只要将 slice 文件夹下的图片替换掉，再通过 Hakon 的处理，重新生成一张新的雪碧图进行替换即可，Hakon 也会在图片的后面加上时间戳，对付浏览器缓存的问题

  2. 使用最新的 css
    这个功能使用的是 postcss 的 cssnext 插件，主要是在使用最新的 css 功能的情况下进行向下兼容，如：

    :root{
     --yel: #ffea00;
     --red: #ff0000;
 }
 h1{
     color: var(--yel);
 }
 h2{
     color: var(--red);
 }

    这是新的 css 变量，俗称 “css4”，但是这种写法现在在很多浏览器中还未实现，当然，除了 chrome ，chrome 一直都是兼容最新的写法，经过 cssnext 的处理后，会生成这样的代码：

    h1{
    color: #ffea00;
}
h2{
    color: #ff0000;
}

    从而兼容所有的浏览器，具体还有其他用途，可以查阅 cssnext 的官网

  3. 其余的功能
    其余的功能基本上没有写法上的不同，在这里不做介绍，大家可以使用 Hakon 处理完，进行代码对比一下，就能发现 Hakon 的神奇之处

Hakon 进行 css 处理的功能，是我现在在工作中使用最多的功能，它帮我完成了很多琐碎的工作，大大提升了工作效率，我不需要管 Hakon 处理完之后的 css，只需要维护我写的源文件，而我能在源文件中使用新的 css 技术、不需要写兼容性前缀等，这样我不需要每次修改都改前缀中的数据，从而也大大便利了之后的维护

模板

模板的功能现在还在改良测试，旨在收集我们常用的代码块（动效、头部尾部等），在线预览，复制使用，界面如下：

我们可以建多个类别，每个类别底下可以有多个模块，模块中可以包含 html、css、js 代码，可以在右下方的区域预览代码效果，从而判断这个模块是否可为我现在在做的项目使用，若可以，则直接复制 html 到我们项目的 html 代码中去，复制 css 到我们的 css 代码中去，js 同理，这样我们就可以便捷地使用这个效果到我们的项目中

模板的功能还在更新完善，之后将做详细介绍
