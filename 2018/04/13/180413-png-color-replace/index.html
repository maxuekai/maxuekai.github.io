几种只使用 css 修改 png icon 颜色的方法

前段时间接到了个需求，做的是嵌在客户端里的 web 页面，而客户端用的就是最新版的 chrome 内核，因此，可以肆无忌惮地使用最新的属性，并且不用太考虑兼容性!

遇到的第一个问题是，项目中用到很多小icon，大小一样，只是颜色有差别，如果需要兼容低版本浏览器，那就只能有几个颜色就切几个颜色的图，但是现在可以随意用最新的属性，所以，有很多种方法可以解决

1. iconfont

一开始是会想到用 iconfont；但是众所周知，iconfont 最大的缺点就是，可维护性差，每增加一个 icon，就得重新生成一遍字体文件

2. fliter: drop-shadow

原理：投影 icon，改变阴影的颜色，之后将原来的元素隐藏掉

这种方法是在张老师的博客里找到的：PNG格式小图标的CSS任意颜色赋色技术
但是比较不好的一点是一个 icon 需要用到两个标签，于是在此基础上进行了实验改造于是有了以下两种方法：

（1） 使用 :before 伪元素

这种方法其实跟张老师的两个元素类似，只是将第二个元素使用伪元素代替

.icon{
    display: block;
    width: 20px; height: 20px;
    overflow: hidden;
}
.icon:before{
    content: '';
    position: relative; left: -20px;
    display: block; width: 20px; height: 20px;
    background: url(../img/icon.png) 0 0 no-repeat;
    filter: drop-shadow(20px 0 #000); // 后面的颜色即要修改成的色值
}

（2） 使用 clip-path 剪裁

既然是将原图案隐藏，自然可以想到使用 cilp 剪裁

.icon{
    display: block;
    width: 20px; height: 20px;
    background: url(../img/icon.png) 0 0 no-repeat;
    filter: drop-shadow(20px 0 #000);
    clip-path: inset(0 -20px 0 20px);
    position: relative; left: -20px;
}

使用 drop-shadow 会有个阴影位移，因此到最后都需要使用定位移动到原来的位置上
这种方法有个很致命的缺点是：
图标的位置和元素的位置不重合，所以这种情况适用于不需要对 icon 进行操作的情况下

（3） 使用 -webkit-mask

mask 可以使用 png 图片做遮罩，使元素只显示图片中有内容的部分，因此只要将元素的背景颜色改为我们所需的颜色，然后用 icon 的图片将透明部分遮挡即可

.icon{
    display: block;
    width: 20px; height: 20px;
    -webkit-mask: urlO(../img/icon.png) 0 0 no-repeat;
    background: #000;
}

于是，我们就可以通过以上的方法，任意改变我们需要的 icon 颜色，而不用切各种颜色的图片了~

以上是现在知道的几种使用 css 修改 png icon 颜色的方法，若有新的方法，将会继续补充~
